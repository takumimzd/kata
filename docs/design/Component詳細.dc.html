使い分け

do & don't 4 例
○推奨
{{ ddDo1 }}
1 画面に primary は 1 つだけ。他は secondary / text で。
{{ ddDo2 }}
破壊的操作は必ず danger + 確認ダイアログを併用する。
×非推奨
{{ ddDont1 }}
primary を並べない。視線が分散して主アクションが伝わらない。
{{ ddDont2 }}
削除など重要な操作を text variant にしない。押しにくく気付きにくい。
